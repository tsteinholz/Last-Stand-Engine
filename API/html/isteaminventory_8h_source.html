138  // RESULT SERIALIZATION AND AUTHENTICATION
139  //
140  // Serialized result sets contain a short signature which can't be forged
141  // or replayed across different game sessions. A result set can be serialized
142  // on the local client, transmitted to other players via your game networking,
143  // and deserialized by the remote players. This is a secure way of preventing
144  // hackers from lying about posessing rare/high-value items.
145 
146  // Serializes a result set with signature bytes to an output buffer. Pass
147  // NULL as an output buffer to get the required size via punOutBufferSize.
148  // The size of a serialized result depends on the number items which are being
149  // serialized. When securely transmitting items to other players, it is
150  // recommended to use "GetItemsByID" first to create a minimal result set.
151  // Results have a built-in timestamp which will be considered "expired" after
152  // an hour has elapsed. See DeserializeResult for expiration handling.
153  virtual bool SerializeResult( SteamInventoryResult_t resultHandle, OUT_BUFFER_COUNT(punOutBufferSize) void *pOutBuffer, uint32 *punOutBufferSize ) = 0;
154 
155  // Deserializes a result set and verifies the signature bytes. Returns false
156  // if bRequireFullOnlineVerify is set but Steam is running in Offline mode.
157  // Otherwise returns true and then delivers error codes via GetResultStatus.
158  //
159  // The bRESERVED_MUST_BE_FALSE flag is reserved for future use and should not
160  // be set to true by your game at this time.
161  //
162  // DeserializeResult has a potential soft-failure mode where the handle status
163  // is set to k_EResultExpired. GetResultItems() still succeeds in this mode.
164  // The "expired" result could indicate that the data may be out of date - not
165  // just due to timed expiration (one hour), but also because one of the items
166  // in the result set may have been traded or consumed since the result set was
167  // generated. You could compare the timestamp from GetResultTimestamp() to
168  // ISteamUtils::GetServerRealTime() to determine how old the data is. You could
169  // simply ignore the "expired" result code and continue as normal, or you
170  // could challenge the player with expired data to send an updated result set.
171  virtual bool DeserializeResult( SteamInventoryResult_t *pOutResultHandle, BUFFER_COUNT(punOutBufferSize) const void *pBuffer, uint32 unBufferSize, bool bRESERVED_MUST_BE_FALSE = false ) = 0;
